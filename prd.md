# Product Requirements Document (PRD)

## 1. Product Overview
**Product Name:** Armaan Choudhary Personal Portfolio
**Theme:** Scrapbook Maximalism / Brutalism
**Objective:** To build a highly engaging, visually distinct personal portfolio website that showcases the user's creative development, design skills, and past projects. It will serve as a primary lead-generation tool for freelance opportunities and full-time roles.

## 2. Target Audience
* Potential freelance clients (agencies, startups, founders).
* Recruiters and hiring managers in tech and design.
* Fellow designers and developers.

## 3. Core Features & Requirements

### 3.1. Homepage (Portfolio Home)
* **Hero Section:** High-impact typography stating the developer's name and core value proposition. Must include a prominent "View My Work" CTA.
* **Visual Style:** Implementation of the dotted/grid background, overlapping "paper" elements, and "tape" visuals.
* **Quick About:** A small snippet ("The Man Behind the Screen") introducing the creator.

### 3.2. Selected Work (Gallery)
* **Grid Layout:** An asymmetrical, scrapbooked grid of past projects.
* **Interactivity:** Hover effects on project thumbnails (e.g., slight rotation, hard shadow expansion, or tape color change).

### 3.3. Project Detail Pages (e.g., Zola Sky System)
* **Dynamic Sections:** Ability to render distinct sections like "The Vision", "Tech Stack", and "Technical Challenge".
* **Media Support:** Support for varied media types—sketches, wireframes, and final polished screenshots, arranged in a collage format.

### 3.4. About & Contact Page
* **Interactive Toolbox:** A visually engaging section listing software and skills using "sticker" or "badge" motifs.
* **Contact Form:** A functional "Say Hello" form capturing Name, Email, and Message. Must include form validation and success/error states.

## 4. Non-Functional Requirements
* **Responsiveness:** The scrapbook aesthetic must degrade gracefully on mobile devices. Elements should stack vertically without losing the brutalist charm (e.g., keeping thick borders and yellow accents).
* **Accessibility:** Despite the chaotic visual style, contrast ratios must meet WCAG AA standards. Form inputs must have clear labels (even if styled uniquely).
* **Performance:** Fast load times (< 2 seconds LCP). Heavy use of optimized imagery is required due to the visual nature of the site.
